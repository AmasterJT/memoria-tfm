En VMware Workstation.
\begin{enumerate}

    \item Acceder al menú superior: \texttt{Edit} $\rightarrow$ \texttt{Virtual Network Editor}

    \item Pulsar sobre \texttt{Change Settings} para habilitar permisos de administrador.

    \item Configurar \texttt{VMnet10} como red interna seleccionando la opción:

    \begin{center}
        \texttt{Host-only} \\
        \texttt{Desactivar \textit{conect a host virtual adapter to his network}} \\
        \texttt{Desactivar el servidor DHCP}
    \end{center}

    \begin{center}
        \texttt{Subnet IP: 10.10.10.0} | \texttt{Subnet mask: 255.255.255.0}
    \end{center}

    \item Aplicar los cambios pulsando \texttt{Apply} y después \texttt{OK}.

\end{enumerate}

Una vez llegamos a este punto, la red interna VMnet10 queda correctamente configurada y lista para ser utilizada por las máquinas virtuales del laboratorio. Ahora falta asignarlas a cada máquian virtual según sea el caso.

Dependiendo del método de despliegue elegido, las máquinas virtuales pueden venir preconfiguradas con la interfaz de red interna VMnet10 o bien será necesario asignarla manualmente durante la instalación.

\subsection*{Métodos de despliegue}

Con el objetivo de facilitar la reproducibilidad del laboratorio, se han definido dos alternativas de instalación:

\begin{enumerate}
    \item Despliegue mediante máquinas virtuales preconfiguradas.
    \item Instalación manual desde cero.
\end{enumerate}

% ======================================================
\section*{Despliegue mediante máquinas virtuales preconfiguradas}
% ======================================================

Este método permite disponer del laboratorio completamente funcional en pocos minutos. Las máquinas virtuales ya contienen todas las configuraciones y vulnerabilidades implementadas durante el desarrollo del proyecto.

\subsection*{Descarga de las máquinas virtuales}

En primer lugar, deben descargarse las máquinas virtuales correspondientes a:

\begin{itemize}
    \item Ubuntu Server 22.04.
    \item DC01 (Windows Server 2012).
    \item DEV01 (Windows 7).
\end{itemize}

\subsection*{Importación en VMware Workstation}

Una vez descargados los archivos, se procede a:

\begin{enumerate}
    \item Descomprimir el contenido.
    \item Abrir VMware Workstation.
    \item Importar las tres máquinas virtuales.
    \item Verificar la configuración de las interfaces de red.
    \item Encender las máquinas virtuales.
\end{enumerate}

\clearpage
\section*{Instalación manual}
% ======================================================

Alternativamente, la infraestructura puede desplegarse desde cero mediante la instalación manual de cada una de las máquinas virtuales. Este procedimiento permite comprender con mayor detalle la configuración interna del laboratorio y facilita su adaptación a otros entornos.

\subsection*{Creación de las máquinas virtuales}

Antes de comenzar la instalación de los sistemas operativos, se deben crear las máquinas virtuales en VMware Workstation con las características indicadas en la Tabla~\ref{tab:ips_laboratorio} y en el Capítulo 5.

Cada máquina debe conectarse a la red correspondiente:

\begin{itemize}
    \item \textbf{Ubuntu Server:} adaptador NAT mediante VMnet8 y adaptador interno mediante VMnet10.
    \item \textbf{DC01:} adaptador interno mediante VMnet10.
    \item \textbf{DEV01:} adaptador interno mediante VMnet10.
\end{itemize}

% ------------------------------------------------------
\subsection*{Instalación del servidor Ubuntu}
% ------------------------------------------------------

El servidor Ubuntu actúa como máquina víctima principal y aloja el portal web vulnerable. Para su instalación se utiliza Ubuntu Server 22.04 con la siguiente configuración:

\begin{itemize}
    \item 2 vCPU.
    \item 3 GB de memoria RAM.
    \item 40 GB de disco.
    \item Adaptador 1: VMnet8 (NAT).
    \item Adaptador 2: VMnet10 (red interna).
\end{itemize}

\subsubsection*{Configuración de red en Ubuntu}

Una vez instalado el sistema operativo, se configura la interfaz interna con una dirección IP estática dentro de la red \texttt{10.10.10.0/24}.

Para ello se edita el fichero de configuración de Netplan:

\begin{bashbox}
sudo nano /etc/netplan/00-installer-config.yaml
\end{bashbox}

El contenido del fichero debe ser similar al siguiente. Los nombres de las interfaces pueden variar según la configuración de VMware, por lo que deben comprobarse previamente con el comando \texttt{ip a}.

\begin{configbox}
network:
  version: 2
  ethernets:
    ens33:
      dhcp4: true
    ens34:
      dhcp4: false
      addresses:
        - 10.10.10.20/24
\end{configbox}

Después de guardar los cambios, se aplica la configuración:

\begin{bashbox}
sudo netplan apply
\end{bashbox}

Finalmente, se comprueba que la dirección IP se ha asignado correctamente:

\begin{bashbox}
ip a
\end{bashbox}

\subsubsection*{Descarga del laboratorio}

Una vez configurada la red, se clona el repositorio del proyecto:

\begin{bashbox}
git clone https://github.com/AmasterJT/TFM-Ciber-CEU-Jose-Torre.git

cd TFM-Ciber-CEU-Jose-Torre
\end{bashbox}

\subsubsection*{Ejecución del script de despliegue}

A continuación se conceden permisos de ejecución al script de instalación y se ejecuta con privilegios de administrador:

\begin{bashbox}
chmod +x scripts/setup_tfm_lab.sh

sudo ./scripts/setup_tfm_lab.sh
\end{bashbox}

Este script realiza automáticamente las siguientes acciones:

\begin{itemize}
    \item Instalación de Docker y Docker Compose Plugin.
    \item Configuración de usuarios y grupos.
    \item Creación de credenciales simuladas.
    \item Habilitación del acceso SSH.
    \item Preparación del escenario de escalada de privilegios.
    \item Despliegue del portal vulnerable mediante Docker Compose.
\end{itemize}

\subsubsection*{Instalación de Active Directory}

Una vez configurada la red, se instalan los roles necesarios para convertir el servidor en controlador de dominio.

\begin{tcolorbox}[
    enhanced,
    breakable,
    colback=black!3,
    colframe=black!15,
    boxrule=0.2pt,
    arc=2mm,
    left=4mm,
    right=4mm,
    top=2mm,
    bottom=2mm
]
\begin{enumerate}
    \item Abrir el \textbf{Administrador del servidor}.
    \item Seleccionar \textbf{Administrar}.
    \item Pulsar en \textbf{Agregar roles y características}.
    \item Seleccionar \textbf{Instalación basada en características o en roles}.
    \item Elegir el servidor \texttt{DC01}.
    \item Marcar el rol \textbf{Servicios de dominio de Active Directory}.
    \item Aceptar la instalación de las características adicionales requeridas.
    \item Marcar también el rol \textbf{Servidor DNS}, si no se selecciona automáticamente.
    \item Continuar el asistente y pulsar \textbf{Instalar}.
\end{enumerate}
\end{tcolorbox}

\subsubsection*{Promoción a controlador de dominio}

Tras instalar el rol de Active Directory, el servidor debe promocionarse a controlador de dominio.

\begin{tcolorbox}[
    enhanced,
    breakable,
    colback=black!3,
    colframe=black!15,
    boxrule=0.2pt,
    arc=2mm,
    left=4mm,
    right=4mm,
    top=2mm,
    bottom=2mm
]
\begin{enumerate}
    \item En el \textbf{Administrador del servidor}, pulsar sobre la notificación superior.
    \item Seleccionar \textbf{Promover este servidor a controlador de dominio}.
    \item Elegir la opción \textbf{Agregar un nuevo bosque}.
    \item Introducir como nombre de dominio raíz: \texttt{corp.local}.
    \item Configurar la contraseña de restauración de servicios de directorio.
    \item Mantener las opciones por defecto para DNS y NetBIOS.
    \item Revisar la configuración.
    \item Pulsar \textbf{Instalar}.
    \item Esperar a que finalice el proceso y permitir el reinicio del servidor.
\end{enumerate}
\end{tcolorbox}

Tras el reinicio, el servidor \texttt{DC01} quedará configurado como controlador de dominio del dominio \texttt{corp.local}.

\section*{Puesta en marcha}
% ======================================================

Una vez desplegada la infraestructura, se recomienda iniciar las máquinas en el siguiente orden:

\begin{enumerate}
    \item DC01.
    \item DEV01.
    \item Ubuntu Server.
    \item Máquina atacante.
\end{enumerate}

Este orden permite garantizar que los servicios de dominio y resolución DNS estén disponibles antes de iniciar el resto de sistemas.

\subsection*{Comprobaciones básicas}

Antes de comenzar los ejercicios de explotación, se recomienda realizar las siguientes comprobaciones.

\subsubsection*{Comprobaciones desde Ubuntu}

\begin{bashbox}
ping 10.10.10.10
ping 10.10.10.30
\end{bashbox}

\subsubsection*{Comprobaciones desde DEV01}

Desde una consola de Windows se puede comprobar la conectividad con el controlador de dominio:

\begin{configbox}
ping DC01
ping corp.local
\end{configbox}

\subsubsection*{Comprobación del portal web}

Desde la máquina atacante o desde un equipo con conectividad hacia el servidor Ubuntu, se comprueba el acceso al portal vulnerable:

\begin{configbox}
http://10.10.10.20:8080/portal_pyme
\end{configbox}

Tras completar estas comprobaciones, el laboratorio quedará preparado para ejecutar los escenarios de explotación descritos en el Capítulo 6.
